Redis集群搭建

搭建

    ######## Cluster Config Start 
    port 9001
    daemonize yes
    bind 192.168.50.167
    dir ./
    pidfile /var/run/redis_9001.pid
    cluster-enabled yes
    cluster-config-file nodes-9001.conf
    cluster-node-timeout 15000
    appendonly yes
    ######### Cluster Config End

    ##开启主从集群 IP可以改为局域网IP
    ./bin/redis-trib.rb create --replicas 1 127.0.0.1:9001 127.0.0.1:9002 127.0.0.1:9003 127.0.0.1:9004 127.0.0.1:9005 127.0.0.1:9006
  • 集群最少三个节点，并且为了冗余数据不丢失采用主从模式，所以一共6个节点
  • mkdir建立6个redis实例目录，独立的redis.conf文件，端口为9001-9006
  • 开启6个redis实例
  • 使用redis-trib.rb命令建立主从集群，cluster使用ruby，需要合适版本的ruby环境，并且gem install redis

使用

    redis-cli -c -h 192.168.50.167 -p 9001

-c连接上后，cluster nodes命令可以看到6个节点，3个master，3个slave。设置每一个键都会对键名进行CRC16%16383，获得节点索引并且重定向到那个实例的shell，进行SET或者其他命令。
如果主Down了，会将slave会替代为主，提供读写。
如果主Recovery了，会变成Slave。

总结

集群模式提高了性能的情况下，也提高了数据的安全性能，有时间可以仔细看看实现。
