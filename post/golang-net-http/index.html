Golang net/Http

2018-01-17

阅读ProxyTool源码时，补充net/http包的理解。

HTTP SERVER

    package main
    import (
        "net/http"
    )

    func h1(w http.ResponseWriter, r *http.Request) {
        w.Write([]byte("This Is From H1"))
    }

    func h2(w http.ResponseWriter, r *http.Request) {
        w.Write([]byte("This Is From H2"))
    }

    type hhandler struct{}

    func (h *hhandler) ServeHTTP(w http.ResponseWriter, r *http.Request) {
        w.Write([]byte("This Is From H3"))
    }

    func main() {
        mux := http.NewServeMux()
        mux.HandleFunc("/h1", h1)
        mux.Handle("/h2", http.HandlerFunc(h2))
        mux.Handle("/h3", &hhandler{})
        http.ListenAndServe(":1234", mux)
    }

上面的HTTP SERVER代码核心理解几个：

路由管理器(ServerMux)

ServerMux类型解释：

    type ServeMux struct {
        mu    sync.RWMutex  //同步锁
        m     map[string]muxEntry  //存储路由和handler的映射关系
        hosts bool // whether any patterns contain hostnames
    }

    type muxEntry struct {
        explicit bool    //路由是否精确
        h        Handler //请求处理方法相当于控制器，实现了ServeHTTP方法
        pattern  string  //匹配模式
    }

这里是主动创建一个ServeMux路由管理器

    http.NewServeMux()  //返回一个ServeMux

但是也可以使用默认的ServeMux，比如：

    http.HandleFunc("/",handler)

    //下面来自server.go 
    // DefaultServeMux is the default ServeMux used by Serve.
    var DefaultServeMux = &defaultServeMux
    var defaultServeMux ServeMux
    func HandleFunc(pattern string, handler func(ResponseWriter, *Request)) {
        DefaultServeMux.HandleFunc(pattern, handler)
    }

HandlerFunc类型

    type HandlerFunc func(ResponseWriter, *Request)

    // ServeHTTP calls f(w, r).
    func (f HandlerFunc) ServeHTTP(w ResponseWriter, r *Request) {
        f(w, r)
    }

这里HandlerFunc相当于func(ResponseWriter, *Request)的一个别名类型，这个类型拥有ServeHTTP方法，只要把类似的方法转型为HandlerFunc类型，就可以调用它的ServeHTTP方法。

    func NotFound(w ResponseWriter, r *Request) { Error(w, "404 page not found", StatusNotFound) }

    func NotFoundHandler() Handler { return HandlerFunc(NotFound) }

同理，上面的404代码，具有类似签名的函数转型 HandlerFunc(NotFound)，因为转型后拥有ServeHTTP方法，Handler是一个接口类型，其中包含ServeHTTP方法，所以返回为一个Handler。
相当于函数实现了接口的某个方法。

类似的：

    package main

    import (
        "fmt"
    )

    // a定义了一个函数结构，b具有相同的函数结构，转型为a后获得的结果，既有b之前的功能，也有a实现的接口方法

    type a func(name string,age uint8)

    func (fun a) impl(){
        fmt.Println("Same Implemetion")
    }

    func b(name string,age uint8) {
        fmt.Println("INFO:",name,age)
    }

    func main(){
        b("biexiang",21)
        after := a(b)
        after("biexiang",21)
        after.impl()
    }

Handler的处理

首先Handler是一个接口，其中包含ServeHTTP方法

    type Handler interface {
        ServeHTTP(ResponseWriter, *Request)
    }

第一种，ServerMux的一个方法调用：

    mux.HandleFunc("/h1", h1)

    //server.go
    //HandlerFunc将handler(函数)转型为Handler接口类型
    func (mux *ServeMux) HandleFunc(pattern string, handler func(ResponseWriter, *Request)) {
        mux.Handle(pattern, HandlerFunc(handler))
    }

    //将路由和handler的映射关系添加到mux(ServeMux)的map字典中
    func (mux *ServeMux) Handle(pattern string, handler Handler) {
        ...
    }

第二种，直接调用handle方法，省去了第一种在函数内将函数转为接口类型，直接在外面进行转换：

    mux.Handle("/h2", http.HandlerFunc(h2))

第三种，是自定义Handler，可以是AdminHandler或者是IndexHandler，只要实现了ServeHTTP方法，就实现了Handler接口，可以直接处理。

    mux.Handle("/h3", &hhandler{})

Serve监听

http包直接提供了可以外调的方法ListenAndServe，其实是内部Server类型的方法。

    type Server struct {
        Addr           string        // 监听的地址和端口
        Handler        Handler       // 所有请求需要调用的Handler（实际上这里说是ServeMux更确切）如果为空则设置为DefaultServeMux
        ReadTimeout    time.Duration // 读的最大Timeout时间
        WriteTimeout   time.Duration // 写的最大Timeout时间
        MaxHeaderBytes int           // 请求头的最大长度
        TLSConfig      *tls.Config   // 配置TLS
    }

ServerMux其实也实现了ServeHTTP方法，所以Server结构体中的Handler就是指ServeMux，为空的话就为默认的。
当请求进来的时候，首先查找ServeMux中的map映射中是否有响应的路由，有的话调用对应handler的ServeHTTP方法处理，并且返回Response。
