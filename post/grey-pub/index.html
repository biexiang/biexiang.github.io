Nginx 配置灰度发布

2017-10-06

用户登录的时候 根据需求对某一部分的用户设置cookie=vip

Nginx接受请求的时候判断 type == 8eec1e97ac602d3228bad33b61efeaae ？
是的话，转发到指定的新功能服务器。
不是的话，转发到生产服务器。

可以是proxy到多台服务器，也可以proxy到多个端口，也可以proxy到多个不同的网站根目录

问题1：预发布服务器和生产服务器的session不同步
默认使用生产服务器的话，在生产服务器登录，如果是预发布用户，设置cookie，跳转到首页，nginx再次判断请求头有无cookie，存在cookie，跳转到预发布服务器，预发布读取不到用户登录状态，跳转首页，需要重新登录。如果没有设置cookie，则还是生产服务器，不影响。

问题2：退出登录 type cookie也需要清除
如果不清除，每次登录都会选择预发布服务器

upstream  backend  {
  server   backend1.example.com    weight=5;
  server   127.0.0.1:8080          max_fails=3  fail_timeout=30s;
  server   unix:/tmp/backend3;
}

app_01 192.168.31.30:80
app_02 192.168.31.32:80

upstream app_01 {
    server 192.168.31.30:80 max_fails=1 fail_timeout=60;
}
upstream app_02 {
    server 192.168.31.32:80 max_fails=1 fail_timeout=60;
}
upstream default {
    server 192.168.31.30:80 max_fails=1 fail_timeout=60;
}
server {
  listen 80;
  server_name  dev.acagrid.com;
  access_log  logs/www.hi-linux.com.log  main;
  #match cookie
  set $group "default";
    if ($http_cookie ~* "version=V1"){
        set $group app_01;
    }
    if ($http_cookie ~* "version=V2"){
        set $group app_02;
    }
  location / {                       
    proxy_pass http://$group;
    proxy_set_header   Host             $host;
    proxy_set_header   X-Real-IP        $remote_addr;
    proxy_set_header   X-Forwarded-For $proxy_add_x_forwarded_for;
    index  index.html index.htm;
  }
 }
