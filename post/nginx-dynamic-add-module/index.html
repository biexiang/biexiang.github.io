Nginx动态添加模块

2017-12-13

好几次为了一个模块重新编译整个Nginx，其实可以更简单。

步骤

  • 我已经安装了Nginx，nginx -V查看编译参数。

    ➜  bin ./nginx_bak -V
nginx version: nginx/1.12.0
built by clang 8.1.0 (clang-802.0.42)
built with OpenSSL 1.1.0e  16 Feb 2017 (running with OpenSSL 1.1.0f  25 May 2017)
TLS SNI support enabled
configure arguments: --prefix=/usr/local/Cellar/nginx/1.12.0_1 --with-http_ssl_module --with-pcre --sbin-path=/usr/local/Cellar/nginx/1.12.0_1/bin/nginx --with-cc-opt='-I/usr/local/opt/pcre/include -I/usr/local/opt/openssl@1.1/include' --with-ld-opt='-L/usr/local/opt/pcre/lib -L/usr/local/opt/openssl@1.1/lib' --conf-path=/usr/local/etc/nginx/nginx.conf --pid-path=/usr/local/var/run/nginx.pid --lock-path=/usr/local/var/run/nginx.lock --http-client-body-temp-path=/usr/local/var/run/nginx/client_body_temp --http-proxy-temp-path=/usr/local/var/run/nginx/proxy_temp --http-fastcgi-temp-path=/usr/local/var/run/nginx/fastcgi_temp --http-uwsgi-temp-path=/usr/local/var/run/nginx/uwsgi_temp --http-scgi-temp-path=/usr/local/var/run/nginx/scgi_temp --http-log-path=/usr/local/var/log/nginx/access.log --error-log-path=/usr/local/var/log/nginx/error.log --with-http_gzip_static_module --with-http_v2_module

  • 下载Nginx源码包

    wget http://nginx.org/download/nginx-1.12.0.tar.gz

  • 待添加的模块

    wget https://github.com/openresty/echo-nginx-module/archive/v0.61.tar.gz

  • 解压两个包，源码包内重新编译并且–add-module=path/to/mudule

    ./configure --prefix=/usr/local/Cellar/nginx/1.12.0_1 --with-http_ssl_module --with-pcre --sbin-path=/usr/local/Cellar/nginx/1.12.0_1/bin/nginx --with-cc-opt='-I/usr/local/opt/pcre/include -I/usr/local/opt/openssl@1.1/include' --with-ld-opt='-L/usr/local/opt/pcre/lib -L/usr/local/opt/openssl@1.1/lib' --conf-path=/usr/local/etc/nginx/nginx.conf --pid-path=/usr/local/var/run/nginx.pid --lock-path=/usr/local/var/run/nginx.lock --http-client-body-temp-path=/usr/local/var/run/nginx/client_body_temp --http-proxy-temp-path=/usr/local/var/run/nginx/proxy_temp --http-fastcgi-temp-path=/usr/local/var/run/nginx/fastcgi_temp --http-uwsgi-temp-path=/usr/local/var/run/nginx/uwsgi_temp --http-scgi-temp-path=/usr/local/var/run/nginx/scgi_temp --http-log-path=/usr/local/var/log/nginx/access.log --error-log-path=/usr/local/var/log/nginx/error.log --with-http_gzip_static_module --with-http_v2_module --add-module=/Users/poly/Desktop/tmp/garbage/echo-nginx-module-0.61

  • make进行编译，不要进行make install
    make install 会导致配置文件和网站清空

  • 替换./objs/nginx到环境变量中的nginx
    注意软连接

  • 测试使用echo-nginx-module

    location /hello {
    echo "hello, world!";
}

----------

➜  objs curl cjy.com/hello
hello, world!

问题

  • Nginx的权限问题

    nginx: [alert] could not open error log file: open() "/usr/local/var/log/nginx/error.log" failed (13: Permission denied)
2017/12/13 22:36:23 [warn] 41178#0: the "user" directive makes sense only if the master process runs with super-user privileges, ignored in /usr/local/etc/nginx/nginx.conf:1
2017/12/13 22:36:23 [emerg] 41178#0: open() "/usr/local/var/log/nginx/error.log" failed (13: Permission denied)

nginx.conf的user是staff组的poly，这个配置没有问题
poly有执行权限x，但是nginx要求super user
最后发现可以将执行权限改为s，执行者将具有该文件拥有者的权限，相当于临时拥有root权限

    sudo chmod u+s nginx

    -------

    -rwsr-xr-x   1 root  wheel  1064556 12 13 22:10 nginx

之前以为只有RWX权限，不知道还有S和T。
