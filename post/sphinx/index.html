Sphinx初试

2017-10-26

用过阿里的开放搜索，将自己的数据根据字段结构导入，直接使用阿里的api来做搜索业务。 实现估计是elasticsearch或者sphinx。

Mac安装Sphinx

  • 主程序安装：
    直接brew install sphinx –with-mysql
    /usr/local/Cellar/sphinx/2.2.11

  • PHP7扩展安装：
    PECL上下载PHP7的包，make时报错需要libsphinx的库文件。
    于是到sphinx官网上下载2.2.11版本的源文件，解压文件，路径:/api/libsphinxclient
    编译安装libsphinxclient
    再重新编译扩展，加载so文件成功。

创建百万级假数据

建立一个基础的表结构，脚本插入100w条数据，假数据生成使用Composer包Faker。

实测结果

  • 建立索引、开启search服务

  • 100w行user表数据，内存占用情况

毫无疑问，在Mysql字段没有建立索引的情况下，Sphinx的查询肯定比数据库快。而且可以在多个index中模糊查询相关的信息，难怪很多在线密码库都是用Sphinx搭建的。

PHP测试脚本

    <?php 
    $start = microtime(true);
    $cl = new SphinxClient();
    $cl->SetServer('127.0.0.1',9312);
    $cl->SetConnectTimeout ( 999 );
    $cl->SetArrayResult ( true );
    $cl->SetMatchMode ( SPH_MATCH_ALL );
    $cl->setLimits(10,5);

    $res = $cl->Query ( 'Willow', "user" ); 
    $end = microtime(true);
    echo PHP_EOL . ($end - $start)*1000 . PHP_EOL;
    print_r($res);

查询结果返回分页后的数据，主要是相关信息的id，再由id去库中查询详细的信息。
只是sql_query一定要把主键id字段也加上。否则建索引时会被跳过。

    source user
    {
          type          = mysql

          sql_host      = 127.0.0.1
          sql_user      = root
          sql_pass      = be_evil
          sql_db        = sphinx
          sql_port      = 3306  

          sql_query_pre           = SET NAMES utf8          
          sql_query               = SELECT id,username,password,salt,nickname,created FROM users

    }
    index user
    {
            source                    = user
            path                      = /usr/local/var/data/user
            docinfo                   = extern
            ngram_len                 = 1
            ngram_chars               = U+3000..U+2FA1F 
    }
