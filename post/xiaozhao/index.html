xargs命令

如下面例子，xargs将管道前的内容作为后面命令的参数，如果有命令的话，比如cat –helps。

    echo '--help' | xargs cat

1、将输入内容转换为单行空格隔开输入

    cat test.txt | xargs 
    a b c d e f g h i j k l m n o p q r s t u v w x y z

2、定义分界符

    echo "nameXnameXnameXname" | xargs -dX 
    name name name name

3、删除所有图片

    find -name '*.jpg' |xargs rm -r

4、复制内容

    find -name '*.xml' |xargs -i cp {} /home/chaowei/data/jinin/label/

Redis中进行唯一计数

  • 基于Set数据结构，不同于list，它自带排重

    SADD KEY 1 2 3
SCARD KEY

  • 基于bitmap结构，位index存储序号，1代表存在，0代表不存在，1的个数代表基数数量，1MB=100w字节=800w bit，所以1MB可以存储800w个基数访问记录。如果需要统计日活登录用户，只需要建立一个BitMap的键，统计Android登录也可以建立一个BitMap等应用。

    SETBIT KEY 1024 1   //设置某个键上某个偏移量上的值为0/1
SETBIT KEY 64 1
GETBIT KEY 64       //获取某个键上某个偏移量上的值
BITCOUNT KEY        //获取某个键1的个数

其实就是BitMap的实现，很节约资源，但是如何计算1的个数。
1、循环右移直到值为0，如果右移后最后一位为1则余1，统计之。时间复杂度依赖于BitMap最高位1的位置。
2、n&(n-1)这样，主动产生1，这样就减小了无效的操作，时间复杂度为1的个数。
3、提前建立一个索引数字对应1的个数的Map，0(1)的操作效率，但是需要提前预判好，建立一个对照表。{0,1,1…8}(8位的对照表，十进制范围0-256)。
4、log2效率，二分法。

  • 基于HyperLoglog，这是Redis后来新增的数据结构，有误差，但是数据量大的话，可以忽略

    PFADD KEY 1 2 3
PFCOUNT KEY

Redis的各种淘汰算法

Redis内存超出，配置文件中会设置Redis实例的最大内存容量，如果超出，以下处理：
1、LRU删除设置了过期时间的键。
2、LRU删除键。
3、随机删除设置了过期时间的键。
4、随机删除键。
5、删除过期时间最近的键。
LRU是后来添加到数据结构中，一共24bit，16bit记录访问时间，8bit记录访问逻辑次数。

判断链表是否有环

一个快指针，一个慢指针，一起走，如果相遇则有环，如果不相遇即无环。

求树的高度

树的高度，递归求解。

    //假设定义了树结构和Create方法
    func getHigh(Tree *Node) int {
        if Tree == nil {
            return 0;
        }
        left := getHigh(Tree.left)
        right := getHigh(Tree.right)
        if left > right {
            return left + 1;
        }else{
            return right + 1;
        }
    }

拥塞控制

慢开始、拥塞避免、快重传、快回复

  • 慢开始：cwnd(congestion window-拥塞窗口)=1开始，主键翻倍增长，1、2、4、8、16增长，但是增长过程中受限于ssthresh门限值。
    cwnd < ssthresh => 继续慢开始
    cwnd > ssthresh => 拥塞避免算法
    cwnd = ssthresh => 两者都ok

  • 拥塞避免算法：一旦cwnd到大ssthresh值，采取拥塞避免，增长方式改为+1线性增长。一旦出现网络拥塞(收不到ACK确认)，cwnd窗口设置为1，并且ssthresh/=2，并且执行慢开始。

Apache采用php_mod模式运行

  • prefork模式 一个进程对应一个请求 线程安全
  • worker模式 多进程多线程 提高性能但是稳定性下降
  • event模式(2.4) 还是比Nginx性能差点

Nginx，php作为fastcgi独立运行，php挂掉了，不影响Nginx的运行，可以ip+端口运行，同机的话还可以作为socket运行。

使用Upstream进行负载均衡的算法

  • 轮询，根据请求的时间顺序分配到应用服务器。
  • Weight，根据权重，将请求尽可能的分配到性能好的应用服务器上，适用于应用服务器配置不一样。
  • IP_HASH，可以对IP的Hash值分配应用服务器，对于某个地区的使用某一台服务器，这样就避免了多台服务器要同步session的问题。
  • Url_HASH，对于某一类路由采用某一台服务器，增加了缓存的命中率，提升性能可以。
  • fair，根据页面大小和加载时长分配服务器。

PHP生命周期

PHP作为一门胶水语言，首先站在一个扩展的角度来看：

Apache中作为模块加载进来

  • Apache启动时，PHP加载进来执行MINIT(Module Init)，遍历php.ini中加载的扩展，对它们进行初始化，执行扩展的PHP_MINIT_FUNCTION()函数。
  • 路由请求进来前，执行RINIT(Request Init)，PHP将扩展需要的一些变量准备好，再次遍历扩展，执行它们的PHP_RINIT_FUNCTION()函数。
  • Excute php file。
  • 请求结束后，清理现场，RSHUTDOWN(Request shutdown)，PHP_RSHUTDOWN_FUNCTION函数。
  • 当模块被Apache不再加载时，执行PHP_MSHUTDOWN_FUNCTION()函数。

单进程的话，就是这个流程，而Apache的prefork模式下，多进程，每个进程都是这样的流程，进程间互不影响，Apache启动时创建CPU数量的进程(假设)，这些进程MINIT，开始循环进行RINIT，EXCUTE，RSHUTDOWN，直到Apache停止MSHUTDOWN。
至于Apache的worker多进程多线程模式，线程在进程下可以共用一些变量资源。减少了一些重复的操作。

缺页计算

如果CPU通过MMU获取地址，寻址时发现数据不存在，发生缺页中断，导致需要去磁盘中进行IO操作取出数据重新加载到内存。而内存的地址空间是固定的，需要动态的发生数据的变化，为了尽可能的减少缺页的次数。就有了这些算法。
* FIFO，使用队列，先进先出，缺页时替换掉队首的。
* LRU(Least Recetly use)，最近最少使用也就是替换掉长时间不使用的，其实就是替换掉前几次没有使用的元素，将其剔除内存，这个是向前看。
* OPT(Optimal Replacement)，最优替换，和LRU不同，它往后看，目前内存中的元素哪个是最靠后将被使用的被剔除。
