Tcp拆包和粘包

2017-12-17

UDP基于报文传输，头部有2个字节表示报文的长度，所以不会发生拆包和粘包的问题。TCP基于字节流的传输，服务端分不清数据包的界限，所以会有这个问题。

表现形式

Client向Server发送两个数据包

  • 正常情况

  • 粘包情况

  • 粘包+拆包情况

原因

  • 要发送的数据大于TCP发送缓冲区剩余空间大小，将会发生拆包。
  • 待发送数据大于MSS（最大报文长度），TCP在传输前将进行拆包。
  • 要发送的数据小于TCP发送缓冲区的大小，TCP将多次写入缓冲区的数据一次发送出去，将会发生粘包。
  • 接收数据端的应用层没有及时读取接收缓冲区中的数据，将发生粘包。

解决方法

  • 固定包的长度
  • 头部添加数据包的长度
  • 设置边界符号

In Golang

  • 固定长度将长度加入包头

    func Encode(input string) ([]byte, error) {
    var len = int32(len(input))
    var pkg *bytes.Buffer = new(bytes.Buffer)

    err := binary.Write(pkg, binary.LittleEndian, len)
    if err != nil {
        return nil, err
    }

    err = binary.Write(pkg, binary.LittleEndian, []byte(input))
    if err != nil {
        return nil, err
    }

    return pkg.Bytes(), nil
}

func Decode(input *bufio.Reader) (string, error) {
    //读取长度
    slice, _ := input.Peek(4)
    sliceData := bytes.NewBuffer(slice)
    var len int32
    err := binary.Read(sliceData, binary.LittleEndian, &len)
    if err != nil {
        return "", err
    }

    if int32(input.Buffered()) < len+4 {
        return "", errors.New("invalid packet length")
    }

    pack := make([]byte, len+4)
    _, err = input.Read(pack)
    if err != nil {
        return "", err
    }

    return string(pack[4:]), nil
}

Golang知识

Socket编程主要设计到net包、io相关的bufio包

  • listen和listenTcp的区别

    func Listen(net, laddr string) (Listener, error) {
     la, err := resolveAddr("listen", net, laddr, noDeadline)
     if err != nil {
        return nil, &OpError{Op: "listen", Net: net, Addr: nil, Err: err}
     }
     var l Listener
     switch la := la.toAddr().(type) {
     case *TCPAddr:
        l, err = ListenTCP(net, la)
     case *UnixAddr:
        l, err = ListenUnix(net, la)
     default:
        return nil, &OpError{Op: "listen", Net: net, Addr: la, Err: &AddrError{Err: "unexpected address type", Addr: laddr}}
     }
     if err != nil {
        return nil, err // l is non-nil interface containing nil pointer
     }
     return l, nil
  }

  • 服务端流程，使用ResolveTCPAddr解析出规范的地址，listenTcp进行端口绑定，accept监听，一旦连接建立返回一个句柄，开协程进行处理，不阻塞别的连接

  • 客户端流程，同样ResolveTCPAddr获取地址，DialTcp给该地址拨号，拨号成功返回conn句柄，进行读写操作

  • Tcp in Golang主要有几个结构：addr、listener、conn

  • 数据传输都是基于二进制数据流，需要把字符串转换为[]byte

  • 引入自己的包时，公有的方法记得大写

  • bufio.NewReader方法接受所有实现了io.read接口的对象，包括os.stdin和net.conn
