认识RSA

2017-10-06

我知道的

加密算法有对称加密和非对称加密，对称加密是指算法用同一个秘钥进行信息的加密和解密，传输过程中需要传输秘钥，极易被截取，不安全。

而RSA属于非对称加密，非对称加密采用两个秘钥，一个是公钥，一个是私钥。一般来说，创建公钥和私钥，将公钥和信息都传给客户端，客户端使用公钥对它所想传给服务端的信息进行加密，而这段密文只能由私钥解开，所以解释权只有服务端拥有。
而RSA算法采取的办法，是首先选取两个极大的质数(素数)，比如Q和P，计算它们的乘积 N = Q * P,并且计算欧拉函数 M = (Q-1)*(P-1)；并且随机选取一个比欧拉函数计算结果小并且与其互质的数e。(一般选取65537) ；并且计算e对于M的模反元素d。最终(n,e)为公钥，(n,d)为私钥。

变量：
Q P N M e d

我不清楚的

1、模反元素的计算：
　
ed ≡ 1 (mod M)

定义：模反元素是指 e的d倍 被 M除余1
已知e和M，求d
等价于：

    ex + My = 1

使用 扩展欧几里得算法 求出x，也就是d。

欧几里得算法 也就是 辗转相除法，表示为Gcd(a,b) = Gcd(b,a-b)；
扩展欧几里得算法是利用结论，如果a,b存在最大公约数为Gcd(a,b)，则存在 Gcd(a,b) = ax + by；
而在这里，e是比M小并且与M互质的数，所以Gcd(M,e) = 1 = ex + My；

    //a > b             

    //当 b ==0 时         
    gcd(a,b) = a；x = 1,y = 0；           

    //当 b != 0 时            
    gcd(a,b) = ax1 + by1            
    gcd(b,a%b) = bx2 + (a%b)y2          
    //gcd的结果相同，这里互质都是1          
    ax1 + by1  = bx2 + (a%b)y2          
    ax1 + by1  = bx2 + (a-(a/b)*b)y2            
    ax1 + by1  = ay2 + bx2 - (a/b)*b*y2         
    // =>
    x1 = y2         
    y1 = x2 - (a/b)*y2

因为x1、和y1都依赖于x2和y2，可以递归求解出x1、和y1。

    //Golang实现
    func Extend_gcd(a, b int) (int, int) {
        if b == 0 {
            return 1, 0
        } else {
            x, y := Extend_gcd(b, a%b)
            x, y = y, x-(a/b)*y
            return x, y
        }
    }

所以就算别人获取了公钥，知道了n和e，私钥由n和d组成，获取d需要M，而获取M需要q和p，也就是需要将质数乘积n拆分成两个乘数。大整数的因数分解的难度成就了RSA的安全。
所以如果你解决这个数学问题，那么广泛应用的RSA算法将不再安全。Ref: Shor算法

2、传输过程：
m是密文的Int型(字符串可以转Asc或者unicode)
加密数据:
m^e ≡ c (mod n) => GET c

解密数据:
c^d = m (mod n) => GET m

上述式，可由解密带入加密式中证明，但是如果大段文本都需要一个一个字符经过这种运算，计算量会有蛮大。
