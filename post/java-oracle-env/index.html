Java Oracle 环境搭建

2017-12-13

Java + Oracle 课程设计，记录Mac下环境的搭建。

Java

以前Mac是自带Java环境的，Sierra没有了，需要自己下载Java安装包安装。 安装完成后，环境变量软链接：

    ➜  blog ls -al /usr/bin/java
    lrwxr-xr-x  1 root  wheel  74  1 11  2017 /usr/bin/java -> /System/Library/Frameworks/JavaVM.framework/Versions/Current/Commands/java

JDK路径：

    /Library/Java/JavaVirtualMachines/jdk1.8.0_144.jdk

Tomcat

  • 官网下载源码，解压，我放在了/usr/local下
  • chmod +x /usr/local/tomcat/bin/*.sh
  • ./bin/start.sh
  • curl 127.0.0.1:8080 Check下

Intellij IDEA

  • NetBeans出品，官网下载
  • License sever address: http://xidea.online
  • Import Eclipse项目
  • 配置JDK路径

Oracle

oracle一个用户，对应一个库，比如system用户对应system库

  • 安装Docker
  • 配置镜像下载路径：https://www.daocloud.io/mirror#accelerator-doc
  • docker pull alexeiled/docker-oracle-xe-11g
  • docker run -d –name “oracle” -p 2405:22 -p 2406:1521 -p 2407:8080 alexeiled/docker-oracle-xe-11g
  • 以后再打开可以 docker start [container_id]

Navicat

用来连接Oracle，因为sshd好像没开启，其实可以选择NetBeans的DataGrip

项目问题

Java Compile 1.8

  • Go to file -> settings -> Build Execution and Deployment -> Java -> Compiler
  • File -> Project Structure -> Module Settings -> Tab: Source: Language Level
  • File -> Project Structure -> Project(In left pane) -> Project language level

程序包javax.servlet不存在

  • Project Structure -> library -> add java -> choose tomcat/lib/servlet-api.jar

Mac选择路径下的文件

  • Command+Shift+G

MANIFEST.MF

  • src/main/resources/META_INF/

打包成war，发布到Tomcat

  • Project Structure -> 配置Artifacts -> + Web Application Archive -> Output Layouts + Directory Content(Meta_inf的上一级目录) -> ok
  • Build -> Build Artifact -> 右上角启动旁边的汤姆猫 Edit Configuration -> Deploy + -> Server 配置Tomcat -> ok
  • classes Mark as Sources
  • 点击右上角的RUN

总结

Oracle用起来很怪，导入/导出数据都很麻烦。好在Docker很方便，比用win的同学感觉还要方便，容器化环境减轻了开发难度。
